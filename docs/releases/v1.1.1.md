# v1.1.1 — Publish pipeline to GitHub Packages

First release published via the GitHub Packages workflow.

## What changed

- `.github/workflows/publish-npm.yml` replaces the prior `publish.yml`. Targets `https://npm.pkg.github.com` instead of `registry.npmjs.org`. Auths with `GITHUB_TOKEN` instead of a stored `NPM_TOKEN`.
- `.github/workflows/verify-install.yml` chains via `workflow_run` and smoke-tests the consumer install path on every publish.
- `package.json` adds `publishConfig.registry` + `publishConfig.access: public`. Duplicate `license` field removed.
- `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json` realigned to `1.1.1` to match `package.json`. The Codex plugin had drifted to `1.0.0`.

## Drift guards

The publish workflow blocks the run when:
1. Release tag does not match `package.json.version`.
2. `.claude-plugin/plugin.json.version` does not match `package.json.version`.
3. `.codex-plugin/plugin.json.version` does not match `package.json.version`.

The verify-install workflow blocks when:
1. Installed version drifts from the version under test.
2. Tarball contains fewer than 50 files.
3. Any of the required runtime entrypoints is missing (`dist/bin/ibr.js`, `dist/index.{js,mjs,d.ts}`, plugin manifests, `.mcp.json`).

## Consumer install

```
echo "@tyroneross:registry=https://npm.pkg.github.com" > .npmrc
echo "//npm.pkg.github.com/:_authToken=$(gh auth token)" >> .npmrc
npm install @tyroneross/interface-built-right@1.1.1
```

The consumer's `gh` token needs `read:packages`. Refresh with `gh auth refresh -s read:packages`.
