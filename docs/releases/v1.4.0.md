# v1.4.0 - Native layout-fill analysis and npmjs release hardening

Release date: 2026-06-08

## Runtime changes

- Adds native macOS layout-fill / gap analysis for `scanMacOS`.
- Reports `layoutFill[]` findings and mirrors them into `issues[]` as `layout-fill:` warnings.
- Adds the Swift extractor `--analyze-layout` mode.
- Ships `LayoutProbe.swift` and `RenderSwiftUI.swift` templates under `assets/native/swift-templates/`.

## Package changes

- Publishes `@tyroneross/interface-built-right@1.4.0` to npmjs and GitHub Packages from GitHub Release `v1.4.0`.
- Keeps `package.json`, `package-lock.json`, `.claude-plugin/plugin.json`, and `.codex-plugin/plugin.json` aligned at `1.4.0`.
- Excludes SwiftPM `.build` caches from the npm tarball by packaging only the sim-driver source files.
- Verified npm v11 package contents: 144 files, 2.4 MB packed, 10.6 MB unpacked, no `.build` cache paths.

## Release workflow changes

- npmjs publish uses OIDC trusted publishing from GitHub-hosted runners.
- Both publish workflows use Node 24, `actions/checkout@v6`, `actions/setup-node@v6`, and disabled package-manager cache.
- GitHub Packages install verification also uses Node 24 and setup-node v6.
- Required npmjs trusted-publisher settings are documented in `docs/PUBLISHING.md`, including `Allowed actions: npm publish`.
