# Web Design References

Reference files for IBR web design planning. Load these from `web-design-router` or `design-director` only when the current build needs more detail than the skill body provides.

## Files

| File | Use |
|---|---|
| `0_router.md` | Classify the web archetype and choose defaults |
| `1_information_architecture.md` | Page hierarchy, navigation, content priority |
| `2_interaction_states.md` | Controls, forms, loading, errors, empty states |
| `3_visual_system.md` | Typography, spacing, grouping, surfaces |
| `4_content_states_voice.md` | UI copy, status, error, empty-state patterns |
| `5_data_visualization.md` | Chart-worthiness, chart routing, metric display |

## Operating Rule

Design guidance should narrow implementation choices. If a reference adds options but does not reduce ambiguity, record the open choice in `design-intent.json` and ask the user only when the choice changes behavior, navigation, or validation.
